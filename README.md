## 自学理由
2026年3月左右开始接触 vibe coding相关内容，尝试制作了人生第一个网站。  
然而这时候问题来了，我看不懂代码，也不懂网页前后端设计，以至于从头到尾都是两眼一抹黑，哪里出了bug都不知道从何改起，而且任何一点小操作都需要反复戳AI，浪费了很多token。  

#### 最后问题怎么解决的？

#### 好了有点扯远了，回来继续讲故事。
接下来我又接触了rime输入法引擎和与之相关的好几款输入法：小狼毫、hamster、元书、小企鹅，没有别的，因为我就这几台设备好玩。   
接触rime的原因也非常简单：其它输入法在隐私方面大多让我有点介意，而且我有些非常个人化的手癖和自造词输入需求，那必须选择我们高度自由化的rime啦wwwww  

也因此不可避免地接触到了yaml语言。  
还有元书输入法需要的jsonnet和cskin皮肤包。虽然官方表示jsonnet并非必须，但是我连着翻了好几个大佬制作的皮肤文件，里面都有jsonnet，那没办法了还是慢慢学吧。  
然后非常经典的问题就来了：**我看不懂代码……？！**  
说起来，仓输入法有相当不错的可视化编辑器，我已经用那个做了三个好看的皮肤，是这个网站：https://lost-melody.github.io/hamster-tools/  
到了元书就没怎么见过，这也是为什么我突然就要去k代码。  
已经拿自己认为配色比较大胆且有着独特视觉风格的皮肤戳ChatGPT帮我转换成适配元书的皮肤文件，然后不出所料看见了各种各样的bug，比如文件里没有jsonnet导入进去还是一片空白，又比如把我最爱的按键配置搞没了，还有切换大写之后出现了很多莫名其妙的快捷键（只有外形没有实际效果）。  
总之**AI确实可以快速解决即时需求**，但它跑出来的东西我看不懂，就不会知道生成的东西到底是对是错。  

不过由于本人在校期间多少学过一点点基础的编程知识（比如Python语言），单纯找个能写代码的软件还算可以，从代码的英文词组大致看出这段代码什么目的也不是完全不行……  
而且时间一长就全部忘掉啦:)  

要问我以自己现在这种什么都忘掉了的状态怎么折腾代码，请看前面提到的：vibe coding。
